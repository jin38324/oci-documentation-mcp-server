# AGENTS.md

## Project overview

This repository contains `oci-documentation-mcp-server`, a Python Model Context Protocol
(MCP) server for searching and reading Oracle Cloud Infrastructure documentation.

The server exposes two public MCP tools:

- `oci_search_documentation(search_phrase: str, limit: int, page: int)`: searches Oracle
  Help Center (OHC) page results and returns pagination plus documentation URLs.
- `oci_read_documentation(url: str, start_index: int, max_lines: int)`: fetches one
  `docs.oracle.com` HTML page, converts it to markdown, indexes it by line number, and
  returns a line window as JSON.

Keep the project small. Prefer direct, well-scoped helpers over new framework layers.

## Technology choices

- Package and environment manager: `uv`.
- Runtime: Python 3.10+.
- MCP framework: `mcp.server.fastmcp.FastMCP`.
- HTTP client: `httpx`.
- Data validation: `pydantic` v2.
- HTML extraction and markdown conversion: `beautifulsoup4` and `markdownify`.
- Logging: `loguru`, controlled by `FASTMCP_LOG_LEVEL`.
- Build backend: `hatchling`.
- Static and test tooling configured in `pyproject.toml`: `ruff`, `pyright`, `pytest`,
  `pytest-asyncio`, `pytest-mock`, `bandit`, and `commitizen`.

Use `uv` commands for dependency and lock-file maintenance.

## File map

- `oci_documentation_mcp_server/server.py`: MCP server entrypoint, FastMCP setup, tool
  registration, CLI argument handling, URL validation, and high-level tool error handling.
- `oci_documentation_mcp_server/ohc_search.py`: OHC Search API client, request parameter
  validation, pagination/result models, and search response normalization.
- `oci_documentation_mcp_server/util.py`: documentation page fetching, HTML-to-markdown
  conversion, line-window formatting, table-of-contents extraction, related-link
  extraction, and in-process document cache.
- `oci_documentation_mcp_server/models.py`: shared public Pydantic models.
- `tests/test_mcp.py`: live/integration tests for MCP tool functions. These tests access
  real Oracle documentation and OHC search endpoints.
- `tests/REST API doc.md`: local reference document for the OHC Search API.
- `README.md`: user-facing overview and MCP configuration examples.
- `pyproject.toml`: package metadata, runtime dependencies, script entrypoint, and tool
  configuration.
- `uv.lock`: locked dependency graph generated by `uv lock`.

## Search implementation notes

- Search uses the public OHC pages endpoint:
  `https://docs.oracle.com/apps/ohcsearchclient/api/v1/search/pages/`.
- Keep the MCP search interface stable unless the user explicitly asks for an API change.
- Model request parameters with Pydantic. The search client should validate `q`, `size`,
  `pg`, `lang`, and optional product filtering before sending requests.
- Parse only the fields needed by the tool response: pagination metadata plus result
  `url`, `title`, and `description`.
- Empty search results should return an empty results list. Network, HTTP, JSON, and
  validation failures should become clear errors that `server.py` reports through the MCP
  context.

## Documentation reading notes

- `oci_read_documentation` only accepts `docs.oracle.com` URLs ending in `.htm` or
  `.html`.
- `start_index` is a 0-based line number, not a character offset.
- The read tool returns JSON with:
  - `stats`: `total_lines`, `total_words`, `start_line`, `returned_lines`,
    `remaining_lines`, and `remaining_words`.
  - `content`: markdown text for the requested line window.
  - `table_of_contents`: only when `start_index == 0`; extracted from markdown headings
    with `level`, `title`, and 0-based `line_number`.
  - `related_links`: only when `start_index == 0`; shaped as
    `{ "page_url": "...", "links": [{ "title": "...", "href": "..." }] }`.
- `related_links.page_url` is the page directory URL, not the full document URL. If the
  source URL ends with `.html` or `.htm`, strip the final path element. For example,
  `https://docs.oracle.com/en-us/iaas/Content/generative-ai/models.htm` becomes
  `https://docs.oracle.com/en-us/iaas/Content/generative-ai/`.
- Do not join `page_url` and `href` in the returned JSON. The caller can combine them when
  needed.

## Cache behavior

- The documentation cache is process-local and stored in `util.py`.
- Default TTL is 24 hours. Default maximum cache size is 128 entries.
- Cache keys are normalized full document URLs.
- Cache values keep only one body representation: `lines`. Do not store both markdown
  text and line lists.
- Cache values also store `source_url`, `base_page_url`, `table_of_contents`,
  `related_links`, `created_at`, and `last_accessed_at`.
- On read, use a fresh cache entry if available. On miss or expiry, fetch the page,
  convert it, extract metadata, and refresh the cache.
- Before inserting, remove expired entries. If the cache is still full, evict entries by
  oldest `last_accessed_at`.

## Maintenance rules

- Ask for confirmation before adding new production dependencies.
- Use `apply_patch` to modify files whenever possible.
- Read files with UTF-8 encoding, for example:
  `Get-Content -LiteralPath .\README.md -Encoding UTF8`.
- Use `-LiteralPath` in PowerShell when paths may contain characters such as `[]`.
- If you need to understand a package, search official documentation first. Only inspect
  package source when documentation is unavailable or insufficient.
- If repository or documentation inspection shows the issue is under-specified or too
  ambiguous to solve safely, stop and ask the user for more information.
- Analyze requirements and existing code from first principles. If existing code is
  unreasonable, optimize it directly instead of patching around it.
- Do not revert unrelated user changes. This repository may have a dirty working tree.

## Common commands

- Refresh the lock file after dependency metadata changes: `uv lock`.
- Run the MCP server module locally: `uv run python -m oci_documentation_mcp_server.server`.
- Run the server over SSE: `uv run python -m oci_documentation_mcp_server.server --sse --port 8888`.
- Run static checks when needed: `uv run ruff check .` and `uv run pyright`.
- Run live MCP tests when network access is available: `uv run pytest tests/test_mcp.py -m live`.
